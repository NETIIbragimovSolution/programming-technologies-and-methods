Документация VirtualMemoryArray.js

Описание каждой функции модуля — что она делает словами.


Вспомогательная функция (вне класса)


createPageStruct(pageFullSize)

Что делает: Создаёт структуру одной страницы в памяти. Возвращает объект с полями: номер страницы в файле (absolutePageNumber, изначально -1), флаг изменения (modified), время последнего обращения (lastAccessTime), буфер битовой карты занятости слотов (bitmap) и буфер данных страницы (data). Размер данных вычисляется как pageFullSize минус размер битмапа. Используется для инициализации слотов буфера страниц.


Конструктор класса


constructor(filename, size, type, strLength = 0)

Что делает: Создаёт виртуальный массив, хранящийся в файле. Принимает имя файла, размер массива, тип элементов ('int', 'char' или 'varchar') и для строковых типов — длину строки. Проверяет, что для char и varchar задана положительная длина. Вычисляет раскладку страниц, открывает или создаёт файлы, создаёт буфер страниц и загружает в него первые страницы.


Внутренние методы (приватные)


_computeLayout()

Что делает: Вычисляет параметры раскладки данных в зависимости от типа. Для int — размер заголовка, число элементов на странице, размер данных страницы и полный размер страницы (битмап + данные). Для char и varchar — то же самое с учётом длины строки и, для varchar, отдельного файла данных. В конце вычисляет общее количество страниц по размеру массива.


_openOrCreateFiles()

Что делает: Открывает основной файл: если он существует — в режиме чтения и записи, иначе создаёт и открывает в режиме записи. Если файл новый — инициализирует его как swap-файл, иначе читает заголовок. Для типа varchar дополнительно открывает или создаёт файл с суффиксом .data и запоминает текущий конец файла для записи новых строк.


_initSwapFile()

Что делает: Инициализирует новый swap-файл. Пишет в начало файла заголовок: сигнатуру «VM», размер массива, символ типа (I/C/V) и при необходимости длину строки. Затем записывает пустые страницы (битмап + данные нулями) на всё вычисленное количество страниц.


_readHeader()

Что делает: Читает заголовок из уже существующего файла. Проверяет сигнатуру «VM», восстанавливает размер массива и тип. Для char и varchar при необходимости читает длину строки из заголовка. После этого заново вызывает расчёт раскладки, чтобы все поля класса соответствовали файлу.


_loadInitialPages()

Что делает: При старте загружает в буфер страниц столько первых страниц, сколько помещается (но не больше общего числа страниц). Страницы с номерами 0, 1, 2, … загружаются в слоты буфера 0, 1, 2, …


_getPageOffset(absolutePageNumber)

Что делает: Возвращает смещение в байтах в основном файле, с которого начинается страница с заданным абсолютным номером. Учитывает размер заголовка и полный размер одной страницы.


_flushPage(bufferIndex)

Что делает: Сбрасывает изменённую страницу на диск. Если в слоте буфера с заданным индексом лежит страница с флагом «изменена», собирает из битмапа и данных один буфер и записывает его в файл по смещению, соответствующему номеру этой страницы. После записи снимает флаг изменения.


_loadPage(absolutePageNumber, bufferIndex)

Что делает: Загружает страницу с заданным абсолютным номером из файла в слот буфера с заданным индексом. Читает из файла полную страницу (битмап + данные), копирует их в структуру страницы в буфере, запоминает номер страницы, сбрасывает флаг изменения и обновляет время последнего обращения.


_offsetInPage(arrayIndex)

Что делает: По индексу элемента в виртуальном массиве возвращает смещение (номер слота) внутри одной страницы — то есть индекс элемента относительно начала страницы (от 0 до itemsPerPage - 1).


_readElementFromPage(bufferIndex, offsetInPage)

Что делает: Читает один элемент из страницы, находящейся в буфере по заданному индексу, по смещению внутри страницы. Сначала по битмапу проверяет, занят ли слот; если нет — возвращает null. Для int читает 4 байта из данных страницы. Для char читает строку фиксированной длины и обрезает завершающие нули. Для varchar по смещению в данных страницы находит смещение в .data файле, читает длину строки и саму строку из .data файла и возвращает её.


_writeElementToPage(bufferIndex, offsetInPage, value)

Что делает: Записывает один элемент в страницу в буфере по заданному индексу и смещению внутри страницы. Устанавливает соответствующий бит в битмапе (слот занят). Для int пишет число в данные страницы. Для char копирует строку в буфер фиксированной длины и записывает в данные страницы. Для varchar формирует запись (длина + байты строки), записывает её в конец .data файла, запоминает смещение и пишет это смещение в данные страницы.


Публичные методы


getPageBufferIndex(arrayIndex)

Что делает: По индексу элемента в виртуальном массиве находит или подготавливает страницу в буфере. Определяет абсолютный номер страницы. Если эта страница уже есть в буфере — обновляет время доступа и возвращает индекс слота в буфере. Если нет — по времени последнего обращения выбирает самую «старую» страницу; если она изменена — сбрасывает её на диск, затем загружает нужную страницу в этот слот и возвращает индекс слота. При выходе индекса за границы массива возвращает null.


get(index)

Что делает: Возвращает значение элемента виртуального массива по индексу. Проверяет границы, находит слот буфера с нужной страницей, вычисляет смещение в странице и читает элемент. При ошибке доступа или неверном индексе выбрасывает исключение.


set(index, value)

Что делает: Записывает значение в элемент виртуального массива по индексу. Проверяет границы, находит слот буфера с нужной страницей, вычисляет смещение в странице, записывает значение в страницу и помечает страницу как изменённую, обновляет время доступа.


getAt(index)

Что делает: То же, что и get(index) — альтернативное имя для чтения элемента по индексу.


setAt(index, value)

Что делает: То же, что и set(index, value) — альтернативное имя для записи элемента по индексу.


close()

Что делает: Корректно закрывает виртуальный массив: сбрасывает все изменённые страницы из буфера на диск, закрывает дескриптор основного файла и при наличии — дескриптор .data файла, обнуляет дескрипторы.
